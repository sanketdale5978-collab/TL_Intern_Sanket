CNN Architecture Overview

Convolutional Neural Networks (CNNs) are the backbone of image-based disease detection. They automatically learn spatial features from raw pixel data, making them far superior to traditional hand-crafted feature methods for tasks like skin lesion classification.

LAYER 01
Input Layer
Receives raw image pixels (e.g. 224×224×3 for RGB)
LAYER 02
Convolution Layer
Applies learnable filters to detect edges, textures, and patterns
LAYER 03
Pooling Layer
Reduces spatial dimensions, retaining the most important features (Max Pooling)
LAYER 04
Fully Connected Layer
Flattens features and performs high-level reasoning via dense neurons
LAYER 05
Output Layer
Produces class probabilities via Softmax (e.g. Benign / Malignant)

About the ISIC Dataset

The ISIC (International Skin Imaging Collaboration) 2018 Archive is one of the most widely used benchmarks for automated skin lesion analysis. It contains over 10,000 dermoscopic images across multiple diagnostic categories including Melanoma, Nevus, and Seborrheic Keratosis.

Why balanced datasets matter: In medical AI, class imbalance leads to biased models that over-predict the majority class. By balancing using the RMSE-based method, each class contributes equally to the loss function, resulting in a fair and clinically reliable classifier.

Preprocessing importance: Raw dermoscopic images contain noise, lighting variation, and artefacts (hair, bubbles). Preprocessing steps such as resizing to 224×224, normalization ([0,1] range), and augmentation ensure the model generalizes and does not overfit.

Technical Concepts

Transfer Learning leverages pretrained weights (trained on ImageNet) as a starting point. Instead of training from scratch, only the custom classification head is trained on skin cancer images — drastically reducing training time while achieving higher accuracy.

ImageDataGenerator applies real-time augmentation (rotation, zoom, flip, brightness shift) during training, effectively multiplying the dataset size and improving model robustness.

The Confusion Matrix shows true positives, false positives, true negatives, and false negatives — critical for evaluating clinical reliability. The Classification Report provides precision, recall, and F1-score per class.

Models Used
MODEL 01
MobileNetV2
Lightweight depthwise-separable CNN optimized for mobile/edge devices. Ideal for Raspberry Pi deployment. Uses inverted residuals and linear bottlenecks.
MODEL 02
EfficientNetB0
Scales depth, width, and resolution uniformly via compound coefficient. Achieves state-of-the-art accuracy with fewer parameters than comparable architectures.

Why 3D Printing in AI Hardware Projects?

3D printing enables rapid hardware prototyping — the ability to design and physically manufacture custom enclosures, camera mounts, and device housings for the Raspberry Pi 4 within hours. This eliminates dependence on off-shelf parts and allows tailored form factors for clinical environments.

Autodesk Fusion 360 is an industry-standard parametric CAD tool that integrates 3D modelling, simulation, and CAM workflows. It was chosen for designing the device enclosure because of its accessible learning curve, cloud collaboration, and direct STL export for 3D printing.

The Bambu Lab H2D is a multi-material, high-precision desktop 3D printer supporting fast FDM printing — ideal for producing prototypes with functional mechanical features at the lab scale.

Tools Used
Autodesk Fusion 360 Bambu Lab H2D
